## 1. 项目概述

### 1.1 一句话定位

一个跑在个人电脑上的**自媒体内容生产工作台**：输入一个选题（主题 / 要求 / 素材），针对 **7 大平台 12 条流程**自动产出符合各平台调性的**标题、正文、封面、配图、视频脚本 / 剪映草稿、字幕、文档**；每个生成步骤都能独立绑定**任意引擎**（本机 CLI / 文本 API / 出图 API / 图生视频 API / 网页端），支持自动并行、人工接管、评审打分，最终**导出产物包由人工手动发布**。

### 1.2 解决的核心问题

1. **多平台格式割裂**：同一选题在抖音 / 小红书 / B站 / 公众号 / 视频号 / CSDN 调性与规格完全不同，人工反复改写成本高。
2. **AI 工具分散**：CLI（Claude/Gemini/Codex/Grok/Kimi）、文本 API（DeepSeek/Grok/Kimi）、出图 API（即梦/Seedream/gpt-image）、网页端各有所长，过去靠人肉在多窗口间复制粘贴。
3. **质量不稳定 + 合规风险**：缺统一质检，发出去才发现标题平庸、封面违规、踩了广告法极限词。

### 1.3 非目标（明确不做）

- 不做自动发布（只生成 + 评审 + 导出 + 发布清单提示）。
- 不自研视频渲染（成片在剪映完成，本系统产剪映草稿 / 降级素材包）。
- 不做多用户 / 云端协作，单机单用户、数据不出本机。
- 不做版权检测（仅清单提示）。

## 3. 总体架构

### 3.1 分层

```
┌─────────────────────────────────────────────────────────────┐
│  前端 UI（React + Vite）  apps/web                             │
│  项目 │ 流程工作台 │ 引擎管理 │ 模板管理 │ 产物预览 │ 评分面板      │
└───────────────────────────┬─────────────────────────────────┘
                 HTTP REST + WebSocket（本地 127.0.0.1:8787）
┌───────────────────────────┴─────────────────────────────────┐
│  本地服务（Fastify）  apps/server                              │
│   路由 routes.ts │ 数据层 db.ts(node:sqlite) │ 引擎预设 seed.ts  │
├───────────────────────────┬─────────────────────────────────┤
│  核心引擎  packages/core                                      │
│   Pipeline DAG 编排 │ 调度（并行/信号量/重试）│ 人工卡点         │
│   composePrompt（口吻+铁律+参数注入）│ 步骤派发 runStep         │
├───────────────────────────┼─────────────────────────────────┤
│  引擎适配层 packages/providers（统一 Provider 接口）            │
│   cli(子进程) │ api-text │ api-image │ api-video │ web(Playwright)│
├──────────┬──────────┬─────────────┬──────────────────────────┤
│ 评审      │ 剪映草稿  │ 模板/节点库  │ 共享类型/Prompt 渲染        │
│ review    │ jianying │ templates    │ shared                    │
├──────────┴──────────┴─────────────┴──────────────────────────┤
│  存储：SQLite（node:sqlite，零原生编译）+ 文件系统 workspace/    │
└──────────────────────────────────────────────────────────────┘
       可选外壳：apps/desktop（Electron，进程内起同一个服务）
```

### 3.2 技术选型（已落地）

| 层 | 选型 | 关键理由 |
|---|---|---|
| 包管理 / 语言 | **pnpm workspaces + TypeScript**（monorepo） | 多包共享类型，构建快 |
| 本地服务 | **Fastify** + `@fastify/static` + `@fastify/multipart` | REST + 静态托管 + 素材上传 |
| 实时通信 | **WebSocket**（`/api/ws`） | 步骤状态 / 流式输出实时推送 UI |
| 数据库 | **`node:sqlite`（DatabaseSync，Node ≥ 22.5 内置）** | **零原生编译**，彻底避开 better-sqlite3 在 Windows / Electron 的 ABI 编译坑 |
| 前端 | **React + Vite**（深色主题，纯 CSS） | 轻、快、热更 |
| 桌面壳（可选） | **Electron**（esbuild 打包，进程内拉起服务） | 需要时给非技术用户一个双击即用的壳 |
| 出图 / 叠字 | **sharp** | 原生比例出图后处理、Grok 底图 + 程序叠中文标题 |
| 网页自动化 | **Playwright**（持久化登录态） | 驱动 ChatGPT/Claude/Kimi/豆包网页端 |
| 文档 | **docx** npm 库 | MV / 提示词文档导出 |
| 打包导出 | **archiver** | 产物 ZIP 一键导出 |

> 运行要求：**Node ≥ 22.5**（为了 `node:sqlite` 与 `process.loadEnvFile`）。前端构建产物 `apps/web/dist` 由服务静态托管，故源码改动需 `pnpm build` 后生效（开发期用 `pnpm dev` 热更）。

## 4. 数据模型（SQLite 主要表）

```
projects          选题（id, title, brief_json, created_at）
materials         素材（id, project_id, kind[text/image/video], content/file_path）
pipelines         流程实例（id, project_id, platform, mode, template_id,
                  options_json, status）
steps             步骤实例（id, pipeline_id, def_id, type, name, needs_json,
                  provider_id, status, prompt_template, prompt_rendered,
                  started_at, finished_at, error）
artifacts         产物（id, step_id, version, kind[text/image/file],
                  label, content/file_path, selected）
reviews           评分（id, pipeline_id, target, scores_json, total, verdict,
                  issues_json, suggestions_json）
providers         引擎配置（id, kind, name, config_json, model,
                  max_concurrency, enabled）
prompt_overrides  用户自定义模板（path, content, updated_at）
```

- 产物文件统一存 `workspace/<project>/<pipeline>/<step>/v<N>/`，按版本归档，单步重跑不覆盖旧版本。
- `node:sqlite` 不接受 `undefined` 绑定参数，写库前所有可空字段需归一为 `null` 或 `""`（已踩坑，见第 12 章）。

## 5. 步骤类型（StepType）

引擎按 `step.type` 派发到不同执行逻辑，当前支持 12 种：

| type | 说明 | 产出 |
|---|---|---|
| `title` | 标题候选（默认 5 个，可人工卡点挑选） | 文本数组 |
| `content` | 正文 / 口播稿 / 博文 / 图文文案 | 文本 |
| `cover` | 封面，直连出图 API，原生比例不裁剪 | 图（多候选 + 派生尺寸） |
| `video` | 分镜脚本 → 剪映草稿（+CSV 降级） | 草稿目录 / CSV |
| `review` | 多维评分 + 极限词预检 | 评分 JSON |
| `lyrics` | MV 歌词 + 歌名 | 文本 |
| `image-prompts` | 批量配图提示词（受「图片数量」参数控制） | 文本（N 段） |
| `video-prompts` | 全曲 / 分镜视频提示词 | 文本 |
| `subtitle` | SRT 字幕 | .srt |
| `docx` | 提示词文档打包 | .docx |
| `batch-images` | 按提示词逐张真实出图（单张可重抽/替换） | N 张图 |
| `image-to-video` | **图生视频**：把每张图转成动态片段 | N 段 .mp4 |

## 6. 节点库与流水线（核心扩展点）

为了「以后还会不断加节点」，制作流程不写死在代码里，而是**数据驱动**：

### 6.1 共享节点库 `nodes/*.json`

可复用的步骤定义集中在 `nodes/`，流水线通过三种方式引用：

1. **字符串引用**：`"review"` → 整段套用 `nodes/review.json`。
2. **覆盖引用**：`{ "use": "cover", "promptTemplate": "...", "aspects": [...] }` → 套用节点并覆盖个别字段。
3. **完整内联**：直接写整个 step 对象（一次性的特殊步骤）。

现有节点：`cover`、`batch-images`、`image-prompts`、`image-to-video`、`review`；外加两个配置表 `sizes.json`（比例 → 尺寸预设）、`options.json`（通用参数预设）。

- `aspects: ["9:16","16:9",...]` 是简写，读取时由 `TemplateStore.resolveTemplate()` 展开成 `coverSizesByAspect` + `coverSizes`（来自 `sizes.json`）。展开在读取期完成，行为与手写完全一致。

### 6.2 流水线文件 `pipelines/<id>.json`

每条流水线 = `platform` + `mode` + `steps[]` + `options[]` + `notes[]`。示例（节选 `xhs-note`）：

```jsonc
{
  "id": "xhs-note", "platform": "xiaohongshu", "mode": "note",
  "steps": [
    { "id": "title", "type": "title", "humanGate": true, ... },
    { "id": "content", "type": "content", "needs": ["title"], ... },
    { "use": "cover", "aspects": ["3:4","9:16","1:1"] },
    "image-prompts",
    { "use": "batch-images", "aspects": ["3:4","9:16","1:1"] },
    { "use": "image-to-video", "when": { "imageGen":"real","videoGen":"real" }, ... },
    "review"
  ],
  "options": [ /* 见 6.3 */ ]
}
```

- `needs` 定义 DAG 依赖，无依赖者自动并行。
- `when` 让步骤按参数条件跳过（如未开「真实出图」就不跑批量出图 / 图生视频）。
- `humanGate: true` 标记人工卡点（如标题选定后再继续）。
- `post: "jianying-draft"` 标记后处理（写剪映草稿）。

### 6.3 参数开放（UI 可调）

`options[]` 把节点里原本写死的参数**提到界面**，三种控件：

| type | UI | 例 |
|---|---|---|
| `choices` | 按钮组 | 生成形式、图片出图、图生视频开关 |
| `select` / `aspects` | 下拉 | 画面比例（9:16 / 16:9 / 3:4 / 1:1） |
| `number` | 数字输入框（min/max/step/hint） | 标题字数上限、正文字数、图片数量（MV 画面数量 4–30） |

参数值存 `pipelines.options_json`，并在引擎 `composePrompt` 时注入提示词：

```ts
if (step.type === "title"  && opt.titleMaxLen) prompt += `…标题严格 ≤ ${opt.titleMaxLen} 字`;
if (step.type === "content"&& opt.contentLen)  prompt += `…正文约 ${opt.contentLen} 字`;
if (step.type === "image-prompts" && opt.imageCount) prompt += `…正好生成 ${opt.imageCount} 张`;
```

同时驱动 `when` 跳步与封面 / 批量出图的尺寸选择（`coverSizesByAspect[aspect]`）。

> **扩展一条新平台流程 = 加一个 `pipelines/xxx.json` + 对应 `prompts/平台/模式/*.md`，无需改代码。**

## 7. 引擎适配层（Provider Abstraction）

所有生成能力统一为一个 Provider 接口，**每个步骤独立绑定**，自由混搭、并行。`ProviderKind` 五类：

| kind | 实现 | 用途 | 预设引擎（seed.ts） |
|---|---|---|---|
| `cli` | 子进程调用本机 CLI | 文本生成走订阅/包月 | `cli-claude/gemini/codex/grok/kimi`、`cli-mock` |
| `api-text` | OpenAI 兼容 `/chat/completions`（含 vision 看图） | 评审、看图、文本 | `api-deepseek`、`api-grok`、`api-kimi`、`api-grok-vision`、`api-kimi-vision` |
| `api-image` | `/images/generations`，原生比例出图 | 封面、批量配图、Grok 叠字 | `api-jimeng`、`api-grok-image`、`api-grok-overlay`、`api-mv-images`、`api-gpt-image`、`img-mock` |
| `api-video` | 异步提交 / 轮询的图生视频 | image-to-video | `api-video-jimeng`（doubao-seedance i2v）、`video-mock` |
| `web` | Playwright 持久化登录态 | 网页端混合调用 | `web-chatgpt/claude/gemini/kimi` |

要点：
- **零配置可跑**：`cli-mock` / `img-mock` / `video-mock` 默认启用，不填任何 key 也能端到端跑通看效果（mock 文本是固定模板，见第 12 章常见疑问）。
- **配置走 `.env`**：`seed.ts` 用 `process.loadEnvFile` 幂等读环境变量，填好 key 对应引擎自动启用。密钥只存本地、`.env` 已 gitignore。
- **api-image**：把 `imageSize` 透传给 API 实现**原生比例**（不裁剪）；支持「Grok 出底图 + sharp 叠中文标题」模式（中文渲染更稳）。
- **api-video**：`submit → poll → download`，默认路径 `/contents/generations/tasks`，逐张图提交、轮询成功后下载 `video_url`；mock 模式写占位 .mp4。
- **健康检查 / 网页登录**：`/api/providers/:id/health`、`/api/providers/:id/web-login`（一键开登录窗口，登录态本地复用）。

## 8. 引擎执行与提示词组装（packages/core）

### 8.1 composePrompt（提示词渲染）

1. `{{var}}` 模板引擎渲染（注入 `brief`、`steps.x.selected`、`options.x`、素材等）。
2. 对 `title/content/lyrics` 注入 `PLATFORM_VOICE`（平台口吻）+ `CONTENT_RULES`（干货铁律）。
3. 注入数字参数约束（字数、张数）。
4. `orientationBlock(aspect)`：横版 16:9 / 方形 1:1 / 其余竖版，给出图提示词加构图保护规范（如 `[VERTICAL 9:16]`）。

### 8.2 runStep（步骤派发）

按 `step.type` 分流：`docx → buildDocxArtifact`；`batch-images → runBatchImages`；`image-to-video → runImageToVideo`；`cover` 传 `imageSize` + `overlayText`；其余走对应 Provider。

- `runImageToVideo`：找上游 `batch-images`/`cover` 的主图（label `画面 N` / `原图`），逐张调 `api-video`，上限 `MAX_CLIPS=30`，产物 label「视频片段 N」。
- `runBatchImages` / `rerollBatchImage`：批量出图与单张重抽 / 替换。

### 8.3 调度

- DAG 拓扑执行，无依赖步骤自动并行；每引擎按 `max_concurrency` 信号量限流。
- 状态机：`pending → running → waiting_human → succeeded / failed`。
- 单步**重跑**保留多版本产物；评审不过可「按建议自动重生成」。
- 全自动模式：免人工卡点 + 评审不过自动重生成一轮。

## 9. 评审、剪映、合规

- **评审（packages/review）**：独立引擎打分（hook / platform_fit / clarity / compliance / seo），输出严格 JSON；内置**极限词 / 敏感词**本地规则预检 + LLM 复检双层。
- **剪映（packages/jianying）**：分镜表 → `draft_content.json` 写入剪映草稿目录；格式不兼容时**降级**为「素材包 + 分镜表 CSV」，主流程不阻塞。
- **合规清单**：硬规则程序校验（字数 / 尺寸 / 数量 / 时长）+ 软规则清单提示（诱导关注、外链、二维码、极限词）。发布前逐条勾选确认。

| **M0 骨架** | pnpm monorepo、Fastify 服务、`node:sqlite` 数据层、Provider 接口与共享类型、WebSocket | 起服务、建项目、库表就绪 |

| **M2 DAG 编排** | 拓扑并行、信号量限流、人工卡点、多版本重跑、封面原生比例；上线 2 条流程 | 标题卡点→并行内容/封面→导出 |
| **M3 节点库 + 参数开放** | `nodes/*.json` 引用解析、`pipelines/*.json` 数据驱动、`options` 三类控件注入提示词与 `when` | 加流程只改 json；UI 调参生效 |
| **M4 剪映 + 视频脚本** | 分镜 → 剪映草稿（+CSV 降级）；视频类流程 | 抖音/小红书/视频号视频跑通 |
| **M5 批量出图 + 图生视频** | `batch-images` / 单张重抽替换 / `image-to-video`（api-video 异步轮询） | N 张图 → N 段片段 |
| **M6 评审 + 合规** | 评分模块、极限词预检、按建议重生成、全自动模式 | 评审入库、不过自动重生成 |
| **M7 全平台 + MV + 桌面壳** | 补齐 12 条流程、MV（歌词/字幕/docx）、Electron 打包、导出 ZIP | 7 平台 12 流程齐活 |

> 每个里程碑都配端到端冒烟测试（`scripts/smoke.mjs`，8 场景）+ `pnpm typecheck`（8 包），作为合并门槛。

## 12. 关键风险、踩坑与对策

| 项 | 等级 | 对策 / 已知坑 |
|---|---|---|
| SQLite 原生编译（Win/Electron ABI） | 高→已解 | 改用 `node:sqlite`，零编译。注意：**不接受 `undefined` 绑定**，写库前归一为 `null`/`""` |
| 网页端风控 / DOM 改版 | 中 | 有头模式 + 人类化间隔；选择器配置化；失败降级「人工复制」 |
| CLI 输出格式不稳 | 中 | 优先 JSON 输出 + 容错解析 + 失败展示原文 |
| 剪映草稿格式非公开 | 中 | 按版本封装；不兼容降级「素材包 + 分镜 CSV」 |
| 出图中文渲染差 | 中 | 默认即梦；可选「Grok 底图 + sharp 叠字」 |
| 账号安全 | 中 | 登录态 / key 仅本地；`.env` gitignore；数据不出本机 |

**常见疑问 —— Mock 标题为什么千篇一律？** 演示文本引擎（`cli-mock` / `scripts/mock-llm.mjs`）的标题是**写死的模板字符串**（既非正则、也非真模型），只为零配置演示。换成真引擎（Claude CLI / DeepSeek API 等）即按提示词由模型真实生成。
