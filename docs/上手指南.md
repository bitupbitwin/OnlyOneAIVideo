# 上手指南（从 git pull 到出成品，照着做就行）

> 面向 Windows。整个过程：拉代码 → 装依赖 → 启动 → 填 key → 用。
> 先不填任何 key 也能跑通（演示引擎），熟悉流程后再填真实 key 出真东西。

---

## 第 0 步：环境要求（只需确认一次）

打开命令行（按 `Win+R`，输入 `cmd` 回车），检查 Node 版本：

```
node -v
```

- 显示 `v22.5` 或更高（含 v24）→ ✅ 可以
- 低于 v22.5 或提示找不到 → 去 https://nodejs.org 下载 **22 LTS** 安装，重开命令行再试

确认 pnpm：

```
pnpm -v
```

- 有版本号 → ✅
- 找不到 → 运行 `npm i -g pnpm` 安装

---

## 第 1 步：拉取最新代码

进入项目目录（按你的实际路径）：

```
cd /d I:\developapp\Auto-media-product
git pull
```

---

## 第 2 步：装依赖 + 构建 + 启动

```
pnpm install
pnpm build
pnpm start
```

看到这一行就成功了：

```
自媒体内容工作台已启动: http://127.0.0.1:8787
```

> 启动时出现 `SQLite is an experimental feature` 是正常提示，忽略。
> **这个命令行窗口要一直开着**（关掉=关掉软件）。以后再启动只需在项目目录跑 `pnpm start` 一条命令。

浏览器打开 **http://127.0.0.1:8787**。

---

## 第 3 步：先空跑一遍熟悉流程（不填任何 key）

此时系统用的是「演示引擎」，生成的是占位内容/纯色图，目的是让你**先把操作摸熟**。

1. 首页点 **「＋ 新建选题」**：
   - **主题**：随便填，如「新手如何 30 天养成跑步习惯」
   - **📝 我的具体要求**：填你的诉求，如「风格活泼点、面向新手、多举例」
   - 点 **「创建项目并上传素材 →」**
2. 进入项目页，可选地在 **「🗂 我的素材」** 区：粘贴一段文字 / 上传图片 / 上传未剪辑视频（演示阶段可跳过）。
3. 往下，**勾选一个平台**（如「小红书 · 图文笔记」），点 **「⚡ 创建并全自动生成」**。
4. 进入流程页，看着它自动跑：标题 5 选 1 → 内容 → 封面 3 张 → 评审打分。
5. 跑完后：
   - 标题区点选你要的那条，封面区点选你要的那张（**点一下即选中**）
   - 底部 **「📋 发布注意事项」** 逐条核对
   - 右上角 **「📦 导出产物包」** 下载一个 ZIP（文案 + 多尺寸封面 + 评审报告）

熟悉了这套操作，就可以填真实 key 出真东西了。

---

## 第 4 步：填真实 key（出真内容的关键）

### 4.1 建 .env 文件

在项目根目录（就是有 `package.json` 的那层），把 `.env.example` 复制一份改名 `.env`：

```
copy .env.example .env
```

### 4.2 用记事本打开 .env 填写

右键 `.env` → 用记事本打开。**填哪几项就启用哪几个引擎，留空的自动停用。**

**最小可用组合（推荐先填这 3 个就够出「文字+中文封面+评审」）：**

```
AMP_CLI_CLAUDE=claude          # 标题/内容，用你的 Claude 订阅（包月不额外花钱）
ARK_API_KEY=粘贴你的火山方舟key   # 封面（即梦，中文最佳），去 https://console.volcengine.com/ark 拿
DEEPSEEK_API_KEY=粘贴你的key     # 评审打分，去 https://platform.deepseek.com 拿
```

**做视频流程再加：**
```
AMP_CLI_GEMINI=gemini          # 生成分镜 → 剪映草稿
```

**要"看图"能力（封面评审/理解你上传的图）再加：**
```
GROK_API_KEY=粘贴你的grok key    # 去 https://console.x.ai 拿；启用 Grok 分镜出图及「底图+程序叠字」（API 按量计费）
```

填写要点：
- `AMP_CLI_*` 填**命令名**（你在命令行能直接敲的那个，如 `claude`）。不在 PATH 就填绝对路径如 `C:\tools\claude.exe`。
- `*_API_KEY` 直接粘贴密钥，**不要加引号**。
- 不用的行删掉或留空。
- 保存文件。

### 4.3 重启使其生效

回到命令行窗口，按 `Ctrl+C` 停掉，再 `pnpm start`。

### 4.4 验证引擎通了

浏览器 → 顶栏 **「引擎管理」** → 对你填过的引擎点 **「检测」**，显示绿色文字＝通了。
（CLI 若检测失败，多半是命令名不对或该 CLI 没装/没登录；API 失败多半是 key 错或没充值。）

> 详细的每项说明、去哪拿 key、以及不想用 .env 改在网页里填，见 [引擎配置指南.md](引擎配置指南.md)。

## 第 5 步：正式生产（填好 key 后）

填好 key 后，每个平台每步已经自动绑好引擎（标题/内容→Claude，封面→即梦，分镜→Gemini，评审→DeepSeek）。正式流程：

1. **新建选题**：认真填主题 + 我的具体要求；把你的原始素材（笔记/图/未剪视频）都传上去——**素材越足，生成越贴合你**。
2. **选平台**：可多选（如同时勾抖音视频 + 小红书笔记 + CSDN），点「⚡ 创建并全自动生成」，多平台并行跑。
3. **挑成品**：标题 5 选 1、封面 3 选 1；看评审打分和建议。
4. **不满意某一步**？两个办法：
   - 点该步的 **「按建议重新生成」**（评审下方）让它自动改
   - 或点 **「✍️ 我自己做」**：复制提示词 → 去 GPT/Gemini 手动生成最好的 → 粘贴文本 / 上传封面图回填 → 后续评审照常
5. **导出**：右上「📦 导出产物包」拿到 ZIP。
6. **发布**（手动）：按导出包里的「发布注意事项」清单，去各平台 App/网页**手动上传发布**（本工具不自动发布，规避封号风险）。
   - 视频流程：导出包里有「剪映草稿」目录，按里面的 README 复制到剪映草稿文件夹，打开剪映精修后导出成片；你上传的原片已放在草稿的 `source/` 子目录。

## 常见问题

| 现象 | 处理 |
|---|---|
| `pnpm install` 报 `ERR_PNPM_MINIMUM_RELEASE_AGE` 之类 | 仓库已配好策略，正常 `git pull` 后不会再有；若有，删 `node_modules` 重装 |
| 启动报 `Could not locate the bindings file` | 那是旧版本的 better-sqlite3 问题，现已移除；删 `node_modules` 重新 `pnpm install` 即可 |
| 端口 8787 被占用 | 用 `set PORT=9000 && pnpm start` 换端口 |
| 某步骤红色失败 | 卡片上有错误原因；改对引擎配置后点「重跑」，不影响其他步骤 |
| 改了 `.env` 不生效 | 要 `Ctrl+C` 停掉再 `pnpm start` 重启 |
| 想换某步用别的引擎 | 流程页该步右上角下拉框直接切，立即生效 |
| 想改生成风格/规范 | 顶栏「模板管理」在线改提示词，保存即生效 |
| 想要独立桌面窗口 | `pnpm --filter @amp/desktop dev`（首次下载 Electron 约 100MB） |

```
git pull → pnpm install → pnpm build → pnpm start → 开 localhost:8787
先空跑摸熟 → 复制 .env.example 为 .env 填 3 个 key → 重启 → 正式全自动生产 → 挑选/接管 → 导出 → 手动发布
```
