# 自动化优化路线图

> 目标：**全自动内容产线**——选题进去，各平台的标题 / 内容 / 封面 / 视频成品出来，人只做挑选与把关。
> 本文是对现状的全局分析与优先级排序，随实现进度更新。

---

## 现状：自动化程度盘点

```
选题录入 ──► 标题×5(自动排序) ──► 内容 ──► 封面×3 ──► 分镜 ──► 剪映草稿 ──► 评审闭环 ──► 导出ZIP ──► 手动发布
   人工        ✅全自动           ✅      ✅并行     ✅      ⚠️半自动      ✅自动重生成    ✅        人工(刻意保留)
                                                          (成片需进剪映)
```

已具备的自动化能力：

| 能力 | 状态 |
|---|---|
| ⚡ 全自动模式：免人工卡点，标题自动采用推荐度第一候选 | ✅ |
| 评审闭环：不通过自动将意见注入提示词重生成并复评（限 1 轮） | ✅ |
| 多版本择优：标题 5 候选（按推荐度排序）、封面 3 候选，事后可随时改选 | ✅ |
| 一键多平台：勾选 N 条流程「创建并全自动生成」，跨流程并行 | ✅ |
| 无依赖步骤自动并行 + 每引擎并发上限 | ✅ |
| 服务重启中断任务自动置失败可重跑 | ✅ |

**当前产线最大的断点：视频成片。** 文字与图片链路已闭环，视频链路停在"剪映草稿 + 分镜表"，成片仍需人进剪映操作。

---

## 优化项与优先级

### P0 —— 打通"成品视频"最后一公里

| # | 优化项 | 说明 | 价值 |
|---|---|---|---|
| 1 | **TTS 自动配音** | 按分镜表逐镜头调用 TTS（火山/Azure/Edge-TTS），产出配音音频 + 时长回填分镜 | 视频自动化的前置件，口播稿→人声零人工 |
| 2 | **ffmpeg 自动合成初稿成片** | 每镜头画面（出图 API 按 visual 描述生成/用户素材）+ 字幕 + TTS 配音 → 直接合成 mp4；剪映只做精修甚至跳过 | ★ 产线闭环的关键一步 |
| 3 | **图生视频 API 接入** | 分镜画面接可灵/即梦视频/Runway 等生成动态镜头，替代静态图 | 成片质感跃升，成本较高可选 |
| 4 | **封面文字程序叠加模式** | 出图模型渲染中文标题不稳定 → 增加"API 出底图 + 程序叠加标题文字"模式（sharp 合成），文字 100% 正确 | 封面可用率大幅提升 |

### P1 —— 自动化的广度与深度

| # | 优化项 | 说明 |
|---|---|---|
| 5 | **选题批量队列** | 一次录入 N 个选题（或导入 CSV/表格），排队全自动生成——晚上跑、早上挑 |
| 6 | **选题源自动抓取** | 热榜/RSS（抖音热点、微博热搜、B 站热门）定时抓取 → 按你的领域关键词过滤 → 自动入队 |
| 7 | **重生成轮次可配** | 现固定 1 轮，改为"直到总分 ≥ 阈值或最多 N 轮"，阈值/轮次在设置中调 |
| 8 | **内容多版本** | 正文/口播稿也产出 2-3 个不同角度的版本（如干货向/故事向），与标题封面一样择优 |
| 9 | **图集多图自动生成** | 抖音图集/B 站图文目前只自动生成首图，图 2-N 的文案已有但图未出——按逐图文案批量出图 |

### P2 —— 质量与运营沉淀

| # | 优化项 | 说明 |
|---|---|---|
| 10 | **品牌资产与封面模板** | 固定字体/色板/水印/头像，封面走"模板 + 换字换图"而非纯生成，保证账号视觉统一 |
| 11 | **数据回流驱动迭代** | 录入发布后的播放/点赞/完播数据 → 关联到当时的标题/封面选择 → 让"推荐度排序"和评审权重随你的账号数据迭代 |
| 12 | **发布半自动化** | Playwright 打开各平台投稿页并自动填充文案/上传文件，**人只点最后的"发布"按钮**——兼顾效率与账号安全（全自动发布仍不做） |
| 13 | **敏感词词库扩充与自定义** | 词库 UI 化维护，按平台分库 |

### P3 —— 工程与稳定性

| # | 优化项 | 说明 |
|---|---|---|
| 14 | 网页端适配器实测校准 | ChatGPT/Claude/Kimi/豆包真实页面选择器需在本机首跑时校准一次 |
| 15 | Electron 安装器实测 | `pnpm --filter @amp/desktop dist` 在 Windows 实机验证 |
| 16 | 失败自动重试 | 引擎调用失败（网络抖动/超时）自动重试 1 次再判失败 |

## 建议的推进顺序

1. **先跑起来用**：用真实引擎（Claude CLI + 即梦）日常生产文字+封面，验证 Prompt 质量并按你的领域调模板——这是性价比最高的一步。
2. **P0.4 封面叠字模式**（小改动，立竿见影）→ **P0.1 TTS** → **P0.2 ffmpeg 成片**：视频链路逐段闭环。
3. **P1.5 批量队列**：量产能力。
4. 数据积累一个月后做 **P2.11 数据回流**。
