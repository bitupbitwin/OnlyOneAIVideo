# 引擎配置指南（填 key / CLI，看这一篇就够）

> 目标：填好密钥后，每个平台的每一步自动绑定到推荐的引擎，直接全自动跑。

---

## 一、最快路径：填一个 `.env` 文件

1. 在项目根目录，把 `.env.example` 复制一份改名为 `.env`：
   ```
   copy .env.example .env        （Windows）
   cp .env.example .env          （Mac/Linux）
   ```
2. 用记事本/编辑器打开 `.env`，按下面说明填写（**填哪几项就启用哪几个引擎，留空的自动停用**）。
3. 保存后重启：`pnpm start`。启动时会自动读取 `.env`。
4. 打开网页 → 顶栏「引擎管理」→ 对填过的引擎点「检测」，绿色就是通了。

> `.env` 已在 `.gitignore` 里，**不会被提交**，key 安全留在你本机。

---

## 二、`.env` 各项填什么

### CLI（走你的订阅，包月、跑文字不额外花钱）—— 文字活儿主力

| 变量 | 填什么 | 对应引擎 |
|---|---|---|
| `AMP_CLI_CLAUDE` | `claude`（Claude Code 的命令名） | Claude Code CLI |
| `AMP_CLI_GEMINI` | `gemini` | Gemini CLI |
| `AMP_CLI_CODEX` | `codex`（走 ChatGPT 订阅） | Codex CLI |
| `AMP_CLI_GROK` | `grok` | Grok CLI |
| `AMP_CLI_KIMI` | `kimi` | Kimi CLI |

- 值就是你**在终端能直接敲的那个命令名**。在 PATH 里就写命令名；不在就写绝对路径（如 `C:\tools\claude.exe`）。
- 不用的 CLI 那行删掉或留空即可。

### API Key（按量计费）—— 看图 / 出图 / 评审

| 变量 | 哪里拿 | 用途 |
|---|---|---|
| `GROK_API_KEY` | https://console.x.ai | Grok 文本、**看图**、出图、封面叠字（你已充值） |
| `DEEPSEEK_API_KEY` | https://platform.deepseek.com | 评审打分（便宜，推荐） |
| `KIMI_API_KEY` | https://platform.moonshot.cn | Kimi 文本 / 看图 |
| `ARK_API_KEY` | https://console.volcengine.com/ark | **即梦/Seedream 出图**（中文封面最佳，支付宝可充） |
| `OPENAI_API_KEY` | （如以后能充值） | gpt-image-1 出图 |

## 三、填完后，每一步默认用哪个引擎？

流程已经预绑好（在 `pipelines/*.json` 里），**对应引擎启用了就用它，没启用就自动回落到演示引擎**：

| 步骤 | 默认引擎 | 需要填 |
|---|---|---|
| 标题 / 内容 | Claude Code CLI（`cli-claude`） | `AMP_CLI_CLAUDE` |
| 分镜（视频流程） | Gemini CLI（`cli-gemini`） | `AMP_CLI_GEMINI` |
| 封面 | 即梦（`api-jimeng`） | `ARK_API_KEY` |
| 评审打分 | DeepSeek（`api-deepseek`） | `DEEPSEEK_API_KEY` |

> 想让封面走「Grok 底图+程序叠字」（零成本、中文 100% 正确）：填了 `GROK_API_KEY` 后，引擎 `api-grok-overlay` 会自动启用。到流程页把封面步骤的引擎下拉切到它即可（或在 `pipelines/*.json` 把封面的 `defaultProvider` 改成 `api-grok-overlay`）。

> **MV 真实批量出图**：在创建 MV 流程时把「图片出图」选项点成「真实出图」，系统会按图片提示词逐张调出图引擎生成图片（裁切到所选比例）。默认用引擎 `api-mv-images`：填了 `ARK_API_KEY` 即可用（复用即梦）；想用单独账号/模型则填 `.env` 的 `MV_IMAGE_API_KEY`（及可选 `MV_IMAGE_BASE_URL`/`MV_IMAGE_MODEL`）。注意：一首歌会生成 15-30 张图，按量计费。

> 想让**封面评审能看图**：把评审步骤的引擎换成 `api-grok-vision`（已随 `GROK_API_KEY` 启用，config 里 `vision:true`）。

## 四、运行时怎么改绑定（不改文件）

每个步骤卡片右上角有**引擎下拉框**，随时切换该步用哪个引擎，立即生效、可重跑。也就是说 `.env` 决定"有哪些引擎可用"，下拉框决定"这一步用谁"。

---

## 五、不想用 .env？也可以在网页里填

顶栏「引擎管理」→ 找到对应引擎 → 编辑 → 在 config JSON 里填 `apiKey`（API）或 `command`（CLI）→ 勾选「启用」→ 保存。效果一样，存在本机数据库里。

> 注意：如果某项 `.env` 填了值，它会在每次启动时覆盖该引擎的对应字段（`.env` 优先）。想完全用网页管理，就别在 `.env` 里填那一项。

---

## 六、最小可用组合（建议先这样起步）

只填这 3 个就能全自动出「文字 + 中文封面 + 评审」：

```
AMP_CLI_CLAUDE=claude       # 标题/内容（你的 Claude 订阅）
ARK_API_KEY=填你的火山key    # 封面（即梦，中文最佳）
DEEPSEEK_API_KEY=填你的key   # 评审打分
```

视频流程再加 `AMP_CLI_GEMINI=gemini`（出分镜 → 剪映草稿）。
看图评审/图片素材理解再加 `GROK_API_KEY`。
